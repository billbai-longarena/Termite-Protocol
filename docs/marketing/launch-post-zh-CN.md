# 发布文案 — 中文

## 短版

大多数 AI 编码 Agent 一旦会话结束，就会“失忆”。

**白蚁协议 Termite Protocol** 是一套让无状态 AI Agent 跨会话协作的环境协议：Agent 不靠对话协调，而是通过环境接力。

- SQLite 信号
- 原子任务认领
- ≤800 tokens 的 `.birth` 快照
- 像信息素一样跨会话累积的场记忆

目前已经在 6 个真实蚁丘、4 次多模型审计实验、900+ commits 中验证。

如果你也在探索多 Agent 编程，但不想把系统建立在无限增长的聊天上下文上，欢迎试试。

## 中版

我把 **白蚁协议 Termite Protocol** 开源了。

它要解决的问题其实很简单：

AI Agent 天生是无状态的，但软件工程不是。一次会话结束后，上下文消失，下一个 Agent 又要重新摸索项目结构、约定和当前状态。

白蚁协议的做法不是让 Agent 之间说更多话，而是把协调能力放进环境里：

- 信号持久化在 SQLite
- 任务通过原子 claim 认领
- 观察沉淀为跨会话记忆
- 新 Agent 通过 `.birth` 读到动态快照，而不是重读一大份协议文档

比较有意思的是，这个机制对强弱模型混编也有效。
在一次审计实验里，1 个 Codex 牧羊人 + 2 个 Haiku 工人实现了 **96.4% 的观察质量**。

如果你关心多 Agent 编码、长期项目连续性、强弱模型协作，欢迎来拍砖。

## 长版 / 论坛版

这段时间我一直在想一个问题：

**为什么 AI Agent 看起来越来越强，但一到真实项目协作里就总是断档？**

核心原因可能很朴素：

- Agent 的会话是临时的
- 项目的状态却是持续的
- 如果协调完全依赖对话，那每个 Agent 都必须持续携带上下文

这对弱模型尤其不友好。

所以白蚁协议选了另一条路：

- Agent 不互相对话协调
- 它们通过仓库环境接力
- 信号存在 SQLite 里
- 认领通过原子事务完成
- 观察像信息素一样沉积
- 新会话通过 `.birth` 拿到压缩后的现场快照

这个项目不是只有概念图，而是有真实审计材料：

- 6 个生产蚁丘
- 4 次多模型审计实验
- 900+ commits
- A-005 里实现了 96.4% 的观察质量

它适合：

- 多 Agent 并行开发
- 长周期项目
- 强弱模型混编
- 需要审计痕迹的工程场景

它不适合：

- 很小的一次性任务
- 纯探索型研究
- 没有持续上下文需求的小脚本

## 别人问“和别家有什么区别”时的首条回复

很多多 Agent 方案是让 Agent 通过对话协调。

白蚁协议是让 Agent 通过环境协调：

- 仓库里存放信号、记忆和恢复提示
- 新会话感知现场，接着前一个 Agent 的工作继续走
- 协调能力跨越会话边界保存下来
