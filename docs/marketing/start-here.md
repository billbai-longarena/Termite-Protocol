# Start Here / 新访客先看

If you are evaluating Termite Protocol for the first time, open these 3 links in order.

如果你是第一次评估白蚁协议，建议按这个顺序看 3 个链接。

## 1. Understand the problem and fit

- English: `README.md`
- 中文：`README.zh-CN.md`

Read this first if you want to know what the protocol is, what it is not, and why the environment-first model matters.

先用这一步判断项目到底解决什么问题、适不适合你的场景，以及为什么它强调“环境优先”而不是“对话优先”。

## 2. Run the 60-second smoke test

- `QUICKSTART.md`

This is the fastest way to see `.birth`, `BLACKBOARD.md`, SQLite signals, and the first pulse on a fresh install.

这是最快验证 `.birth`、`BLACKBOARD.md`、SQLite signal 和首轮 pulse 是否正常生成的路径。

## 3. Check the latest stable release notes

- `docs/releases/v1.1.1.md`

Use this to confirm what changed most recently, whether the smoke test path is currently safe to recommend, and what success markers maintainers verified.

用这一步确认最近一次稳定发布修了什么、当前 smoke test 是否适合对外推荐，以及维护者验证过哪些成功标记。

## After the first run / 跑完之后

- Compare your output with `docs/marketing/smoke-test-proof.md`
- Ask a setup question or report friction in GitHub Discussions
- If it worked in a real repository, share the repo shape, model mix, and failure modes you observed
