# Protocol Philosophy Revision: From "Blind" to "Stateless + Intelligent Environment"

Date: 2026-03-03
Status: Analysis complete, pre-design
Origin: Dry stone wall metaphor analysis → foundational principle re-examination

> **指导原则：沿袭长期智慧，而不是短期效率和聪明。**

## 背景

通过"堆砌乱石墙"的比喻对照白蚁协议，发现协议存在结构性盲区。逐层追问后，发现盲区的根源不在具体机制，而在协议的基础隐喻——"白蚁是盲的"——存在误导性。

本文档记录三个递进的洞察和 11 个衍生概念的重新澄清。

---

## 洞察 1：白蚁不是盲的，白蚁是无状态的

### "盲"的三层含义

| 层 | 含义 | 对 LLM 成立吗 |
|----|------|---------------|
| 盲-1 | 无跨会话记忆 | **成立** — 基础设施事实 |
| 盲-2 | 只能感知局部信号 | **成立** — context window 有限 |
| 盲-3 | 不理解自己在建造什么 | **强模型不成立** |

### 审计证据

| 实验 | 配置 | 盲-3 成立？ | 结果 |
|------|------|------------|------|
| A-003 ReactiveArmor | 纯弱模型 | 成立 | 9/14 退化观察，0 规则涌现，失败 |
| A-005 touchcli | 1 强 shepherd + 弱模型 | 强模型不成立 | 交接 99%，观察 96.4%，成功 |
| A-006 touchcli | 5 模型异构 | 混合 | 559 commits 高吞吐，57% 观察质量，墙高但脆 |

### 结论

协议的每一个成功案例，都依赖于强模型**违反**盲-3。Shepherd Effect 的本质是强模型**看见了**项目结构，把理解编码进信息素链。

### 替代原则

旧：**"所有白蚁是盲的，通过信息素协调"**
- 推导出：所有白蚁平等，遵循相同规则
- 但强模型实际上在做不平等的贡献
- PE-005 以补丁方式打 strength tiers

新：**"所有白蚁是无状态的，环境承载智慧"**

**原则 A：无状态（不可协商的约束）**
- 任何白蚁可以在任何时候死掉，不丢失系统性信息
- 所有持久知识必须存在环境中

**原则 B：环境承载智慧（设计杠杆）**
- 智慧在信号系统中，不在个体白蚁脑中
- 强模型的核心职责：向环境注入结构化知识
- 弱模型的核心职责：从环境中读取指令并执行

## 洞察 2：11 个衍生概念的重新澄清

### 概念对照表

| 概念 | 旧理解 | 新澄清 |
|------|--------|--------|
| **Stigmergy** | 间接协调 | 错误。协议使用的是环境介质的**显式通信**——强模型写 observation_example 时知道后继者会读。通信质量是一等设计变量。 |
| **信息素 (Pheromone)** | 统一概念：白蚁的痕迹 | 混淆了两种东西：**痕迹**（事实，工具保证可靠：git commit）和**沉淀**（知识，模型能力决定可靠性：observation）。应有不同的信任级别和衰减规则。 |
| **观察 (Observation)** | 所有白蚁都观察模式 | 三种东西：事实报告（任何模型）/ 模式识别（强模型可靠）/ 结构判断（仅强模型）。弱模型的"观察"多数是事实报告或噪声。 |
| **涌现 (Rule 7)** | 3 条相似观察 → 规则 | 有噪声淹没风险。1 强 + 10 弱的蚁丘中，3 条相似观察大概率来自弱模型 → A-006 证明：21 条规则全退化。涌现输入需要区分来源可靠性。 |
| **.birth** | "出生证明" | 实际是**环境对白蚁的指令接口**：工单（execution）、简报（judgment）、仪表盘（direction）。 |
| **Shepherd Effect** | 被发现的部署模式 | 不是建议，是**协议有效的核心机制**。没有它协议退化为机械循环（A-003）。应在 Part I 的地位。 |
| **种姓 (Caste)** | 自主判断的角色 | 对强模型：自觉的形态选择。对弱模型：环境赋予的任务标签。11 级瀑布是强模型概念，弱模型不需要。 |
| **通道特性** | 高带宽 vs 深共振（速度差异） | 不是速度差异，是**功能差异**：生产者（理解 + 写入知识）vs 消费者（读取指令 + 执行）。 |
| **T0/T1/T2** | 连续能力谱 | T0 = 消费者，T2 = 生产者，T1 = **信任区间**（协议在观察输出质量来决定信任度，不是静态能力等级）。 |
| **"盲是优势"** | 盲带来客观性 | 三条优势（不受旧假设污染、无 ego、无自保）全部成立，但原因是**无状态**，不是"看不见"。 |
| **心跳 (Heartbeat)** | 统一节律 | 框架统一（arrive → sense → act → deposit），实现分化。无需修改。 |

## 洞察 3：统一文本取代分类 — "仁者见仁智者见智"

### 问题

"强"和"弱"是相对概念。测试分类有内在缺陷：

| 问题 | 说明 |
|------|------|
| 能力是向量不是标量 | 模型可能代码强、架构弱；Python 强、Rust 弱 |
| 冷启动 | 前 3 次默认 execution，可能浪费强模型 |
| 自我实现偏差 | 简化工单 → 简化产出 → 确认"果然是弱的" |
| 任务依赖 | 同一模型在不同任务类型上表现不同 |
| 模型进化 | 新模型发布后分类阈值全部失效 |

### 两种机制

**机制 A（PE-005 当前）**：测试 → 分类 → 差异化指令
- 到达 → 推断 strength_tier → 生成差异化 .birth → 分发

**机制 B（仁者见仁智者见智）**：统一文本 → 各自提取 → 各自行动
- 到达 → 同一份分层文本 → 能力差异通过提取深度自然表达

### 长期健壮性对比

| 维度 | 测分类 (A) | 统一文本 (B) |
|------|-----------|-------------|
| 分类错误 | 有 | 无 |
| 冷启动 | 3 次观察期 | 即插即用 |
| 模型进化 | 阈值需重新校准 | 自适应 |
| 概念开销 | 3 套模板 + inference 算法 + DB 字段 | 1 套分层模板 |
| 自我实现偏差 | 有 | 无 |
| 边界模型 | 被迫二分 | 自然过渡 |

### 关键证据

A-003 数据：弱模型面对完整协议文本时**没有被搞糊涂**。它们忽略了不理解的部分，只执行了能执行的。观察退化是能力不够，不是信息过载导致混乱。

**弱模型在信息过载时 → 忽略多余的（无害）。弱模型在信息不足时 → 无法获取所需的（有害）。**

因此：提供更多上下文比提供更少更安全。

### 智慧传承

这个原则有长期的验证：

- **自然界**：森林不测试你是鹰还是蚂蚁。同一片森林，鹰看到全景，蚂蚁看到脚下土壤。
- **教育**：好的教科书不出两个版本。入门者读到基础，专家读到方法论。
- **中国哲学**：仁者见仁，智者见智。同一个现实，不同感知取决于观察者自身的能力。
- **API 文档**：Quick Start → Concepts → Advanced，全在一篇文档里。

这些模式跨越了千年，因为它们在参与者多样性面前**无需分类即可运作**。

### .birth 的分层设计方向

```
第一层（表面）：任务 + 行为模板 + 安全网
  → 任何模型都能提取并执行

第二层（中层）：战略上下文 + 规则 + 信号关系
  → 中等以上模型读懂并调整行为
  → 弱模型读到但不理解，忽略（无害）

第三层（深层）：涌现提示 + 蚁丘相位 + 结构判断邀请
  → 只有强模型响应
  → 弱模型无视（无害）
```

层次不是标注为"初学者/专家"的独立段落，而是在自然行文中交织——任务描述中自然包含战略上下文，行为模板中自然包含结构判断的线索。

### PE-005 的保留与废弃

| 保留（洞察） | 废弃（机制） |
|-------------|-------------|
| 不同能力贡献不同 | 预分类到 tiers |
| 行为模板帮助弱模型（Shepherd Effect） | 3 套独立 .birth 模板 |
| 观察质量因模型而异，需要质量门禁 | strength_tier 作为一等 DB 概念 |
| | "observation: optional" 由分类决定 |

## 乱石墙比喻的完整分析（附录）

### 乱石墙工人的六种能力

| 能力 | 协议现状 | 在新原则下 |
|------|---------|-----------|
| 读石（判断材料形状） | 信号只有 type/weight/module，缺少"形状"描述 | 环境应提供信号结构元数据 |
| 读缝（理解缺口形状） | HOLE 信号 + BLACKBOARD 健康状态，较好 | 保留 |
| 试放（低成本试探） | claim 是二元锁，无轻量探测 | 可增加 probe 操作 |
| 邻石感知（依赖关系） | 信号之间无结构化依赖 | 环境应包含信号间关系 |
| 承重判断（结构重要性） | weight = 紧急度 ≠ 结构重要性 | 概念区分 |
| 墙体轮廓（全局结构） | `.field-breath` 只有标量摘要 | 环境应包含架构拓扑 |

### 两个补充洞察

**原材料来源**：协议管理协调循环（信号 → 行动 → 沉淀），不管理材料循环（生成 → 建造 → 使用）。这可能是正确的架构边界（CI/CD 管材料质量，F-010 WONTFIX 是对的）。缺失的是造物的信号可引用性（层1：登记），不是材料验证（层2：CI/CD 范畴）。

**造物信号可达**：白蚁建造的东西（模块、工具、框架）在文件系统中存在，但对信号系统不可见。下一只白蚁到达时不知道"已有一个 retry 工具可以用"。这在"环境承载智慧"原则下是必然需要补齐的——环境必须包含造物信息。

### 张力检查

| 张力 | 诊断 |
|------|------|
| F-009（概念面积膨胀） | 通过数据丰富（JSON 数组）而非概念增加来扩充环境智慧 |
| F-010（结果验证 WONTFIX） | 尊重边界：协议登记造物但不验证 |
| BLACKBOARD 已有原始资产跟踪 | gap 比预想小，需要结构化格式（JSON），不需要新概念 |

### 格式决策

LLM 操作 JSON 比 markdown 表格更可靠（grep/insert/rewrite）。BLACKBOARD 的结构化部分（健康状态、资产索引、信号表）应迁移到 JSON，叙述性内容留在 markdown。

## 批判与盲区（Nurse 审查）

以下是对洞察 3（统一文本原则）的反向审查。洞察 1（无状态取代盲）和洞察 2（概念澄清）基本成立，主要张力集中在"仁者见仁智者见智"的落地可行性。

### 批判 1：800 token 预算与统一文本的矛盾

.birth 有 800 token 硬预算。PE-005 给 execution tier 500 tokens 的聚焦工单。统一分层 .birth 把三层塞进 800 tokens，每层平均 ~267 tokens。

对弱模型而言：500 tokens 的精确工单 vs 267 tokens 的工单 + 267 tokens 它会忽略的战略上下文 + 267 tokens 它会忽略的涌现提示——前者的实际效用可能更高。

**森林比喻的断裂点**：森林是无限的，800 tokens 不是。纸条上写多了，每一层都被稀释。

### 批判 2："忽略"不是弱模型唯一的失败模式

A-003 的证据是弱模型"忽略了不理解的部分"。但 A-006 的数据显示另一种失败模式：`pattern: "HEARTBEAT-PHASE6-START"`, `pattern: "FINAL-HEARTBEAT-PHASE5-SCED"` —— 弱模型不是忽略，而是**捡起协议术语然后错误使用**（cargo culting）。

统一文本让弱模型暴露在更多协议概念面前 → 更多 cargo culting 素材 → 更多格式正确但语义空洞的 deposit。

A-003 的"没被搞糊涂"可能是因为 v3.4 没有丰富的 behavioral template，不是弱模型天然能过滤信息。

### 批判 3：Shepherd Effect 实际上支持分类

"强模型生产环境智慧，弱模型消费"——这个洞察的逻辑终点是知道谁是生产者、谁是消费者**是有用的信息**。

移除 strength_tier + 保留质量门禁 = 在 observation 层面重新做了分类，而不是在 agent 层面。**复杂度没有消失，只是从入口挪到了出口。**

但这个挪动本身可能是有价值的——分类 artifact 比分类 agent 更准确（直接看产出质量，而不是猜测能力）。需要在设计文档中正面承认这个转移。

### 批判 4：涌现修复与统一文本的内在矛盾

文档提议涌现输入按来源可靠性加权。但统一文本原则下没有 strength_tier。加权的依据只能是 observation 本身的内容特征——这就是 artifact 级别的分类，需要设计出具体的质量评分机制。

### 批判 5：自然比喻的感知机制差异

森林里鹰和蚂蚁用**不同的感知器官**自然过滤。LLM 全用同一个 text token processor。弱模型没有"不同的感官"——每个 token 都进了 attention 计算，不理解的内容不是"不存在"，而是"噪声"。

教科书比喻也有断裂：人类读者主动导航（跳章节），LLM 序列处理不能"跳过"。

### 批判 6：可观测性的不对称

PE-005 的分类错误是**可观测、可修复的**（发现强模型被错分，修正阈值）。统一文本的信息稀释是**不可观测的**（无法知道弱模型因为多了 200 tokens 战略上下文而在工单理解上损失了多少精度）。

可观测的缺陷 vs 不可观测的退化——后者可能更危险。

### 综合判断

| 洞察 | 判断 |
|------|------|
| 洞察 1："盲"→"无状态" | **成立**，无争议 |
| 洞察 2：11 个概念澄清 | **基本成立**，trace/deposit 区分尤其锐利 |
| 洞察 3：统一文本取代分类 | **原则成立，机制需修正**：800 token 约束下不能天真地"什么都给"；需要一个既不预分类 agent 又不浪费 token 的中间路线 |

**可能的合成方向**：不分类 agent（保留"仁者见仁"精神），但分类 artifact（在 deposit 时评估质量）+ 让 .birth 内容由**蚁丘状态**驱动（不是由 agent 身份驱动）。详见设计文档。

## 对协议演化的影响

本文档是分析记录，不是设计方案。后续如果要落地到协议 v5.0，影响面包括：

1. TERMITE_PROTOCOL.md Part I 叙述 — "盲白蚁"措辞需修正
2. PE-005 机制 — 差异化 .birth 替换为统一分层 .birth（需解决 800 token 约束）
3. 规则 7（涌现）— 涌现输入需区分来源可靠性（通过 artifact 质量评分，非 agent 分类）
4. Shepherd Effect — 从 Part III 部署建议提升为 Part I 核心机制
5. 信息素概念 — 区分痕迹（trace）和沉淀（deposit）
6. BLACKBOARD 格式 — 结构化部分迁移到 JSON
