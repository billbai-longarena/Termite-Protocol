# Code of Conduct

## Our standard

We want Termite Protocol to be a thoughtful, respectful, and technically rigorous open source project.

Examples of behavior that help build a healthy community include:

- Being respectful in disagreements.
- Assuming good intent while staying precise about technical claims.
- Welcoming newcomers and answering questions with context.
- Giving actionable feedback on code, docs, experiments, and research claims.
- Citing evidence when challenging results or design choices.

Examples of unacceptable behavior include:

- Harassment, personal attacks, or discriminatory language.
- Dismissing others without engaging the substance of their contribution.
- Repeatedly derailing technical threads with unrelated arguments.
- Publishing private information without explicit permission.
- Using the project to spam, mislead, or manipulate others.

## Scope

This code of conduct applies in repository discussions, issues, pull requests, release threads, and other community spaces directly associated with this project.

## Enforcement

Project maintainers may remove comments, close threads, reject contributions, or block participation when behavior harms the project or its contributors.

## Reporting

For community issues, contact the maintainers through GitHub Discussions or a minimal issue that requests moderator attention.

For security-sensitive matters, follow `SECURITY.md` instead of posting details publicly.
